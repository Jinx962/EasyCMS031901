# technology/

本目录用于记录项目的**技术选型与架构决策**，是所有开发工作的技术基准。

## 目录结构

- `TECH_STACK.md` — 技术选型主文档（**开发前必读**）
- `ARCHITECTURE.md` — 当前研发执行的技术架构摘要
- `API_SPEC.md` — 当前阶段接口执行基线
- 可按需追加：`API_SPEC.md`（接口规范）、`DB_SCHEMA.md`（数据库设计）、`DEPLOY.md`（部署方案）等

## 使用规则

1. **开发前必须阅读 `TECH_STACK.md`**，确认项目类型（全栈 / 前后端分离）及各端技术栈。
2. 所有代码必须严格遵循本目录中的技术决策，**不得擅自引入未记录的框架或依赖**。
3. 若需要新增或变更技术选型，必须先更新 `TECH_STACK.md`，同时在 `memory/` 写入变更记录，再开始编码。
