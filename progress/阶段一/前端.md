## 二、本阶段待完成任务（按优先级）

### P0（必须完成）

#### 1) 认证链路稳定化

- [x] 登录失败场景处理：错误提示统一（账号锁定、密码错误、Token 失效）。
- [x] 登出链路统一：主动登出 + 401/1002 被动登出都回到登录页。
- [x] 会话状态校验：应用初始化时拉取 `auth/me`，失败立即清会话。

**验收标准**
- 未登录访问后台任意页面都会跳转登录页。
- 登录后刷新页面不丢失登录态（有效期内）。
- Token 失效后自动退回登录页且不会出现白屏/死循环。

#### 2) 核心页面去 Mock（接口落地）

- [x] 用户管理页：列表、筛选、分页、新建、状态切换全部走后端接口。
- [x] 用户详情页：详情获取、编辑保存、启停账号全部走后端接口。
- [x] 角色管理页：列表、新建、复制、状态切换全部走后端接口。
- [x] 角色详情页：详情、编辑、权限读取、绑定用户读取全部走后端接口。
- [x] 权限配置页：菜单树 + 角色权限读取 + 保存权限走后端接口。
- [x] 审计日志页：列表筛选、详情弹窗走后端接口。
- [x] 菜单管理页：菜单树 CRUD + 状态切换 + 删除走后端接口。

**验收标准**
- 页面不再依赖本地 mock 常量。
- 每个操作均有成功/失败提示，失败时可恢复继续操作。
- 刷新页面后数据一致，不出现“视觉成功但数据未更新”。

---

#### 3) 权限控制（阶段一基础版）

- [x] 菜单权限：根据 `auth/menus` 控制左侧菜单显隐。
- [x] 路由权限：无权限路由显示“无权限/404”而非空白页。
- [x] 按钮权限（基础）：对关键按钮做显隐或禁用（新建/编辑/删除/导出等）。

**验收标准**
- 不同角色登录后，菜单和关键操作按钮表现有差异。
- 受限功能不可操作且提示明确。
