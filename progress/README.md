# progress/ 阅读指引（先看这里）

本目录用于记录项目进度。为了让人快速看懂，建议按以下顺序阅读：

1. `CURRENT_PROGRESS.md`：当前阶段、整体状态、阻塞与下一步（**1 分钟总览**）
2. `阶段N/OVERVIEW.md`：当前阶段完整任务清单（**执行基准**）
3. `阶段N/frontend.md` / `backend.md` / `ui.md` / `tests.md`：各端细节进度（**按角色深读**）
4. `PHASES.md`：全项目阶段规划（**仅在需要看全局路线时阅读**）

---

## 状态标记（统一口径）

- `🟢 已完成`：已交付并通过基本自测
- `🟡 进行中`：正在开发/联调
- `🔴 阻塞`：受外部条件限制无法推进
- `⚪ 未开始`：尚未启动

---

## 维护规则（简化版）

- 每次更新优先改 `CURRENT_PROGRESS.md` 的：
  - 当前阶段
  - 各端状态
  - 阻塞项
  - 下一步计划
- 各端细节写入对应子文件，避免多人改同一文件冲突。
- 阶段切换时，更新 `CURRENT_PROGRESS.md` 的阶段切换记录。
