# memory/

本目录用于记录项目关键变更的**上下文快照**，帮助 AI 或开发者在接续开发时快速还原决策背景。

## 命名规则

- 文件名格式：`YYYY-MM-DD.md`（同一天有多条可用 `YYYY-MM-DD-2.md`）
- 每次发生触发条件时新增一条记录，**不修改已有历史记录**

## 何时必须写 memory（触发条件）

以下情况发生后，**AI 必须主动在 `memory/` 下新增或追加当日记忆文件**：

1. **修改了基础配置文件**：`AI_PRD_GUIDE.md`、`rule.md`（变更了开发规范或项目初始化模板）
2. **修改了需求文档**：`PRODUCT_REQUIREMENTS.md` 发生实质性内容变更（新增/删除/修改功能、验收标准、技术选型等）
3. **变更了技术架构**：技术栈选型、目录结构约定、接口规范、数据库 Schema 发生重大调整
4. **完成重要里程碑**：一个完整阶段交付、上线部署、重大功能 GA
5. **重大 Bug 修复或架构重构**：影响多模块的修复或方案重写

## 每条记忆文件内容格式

```md
# 记忆 YYYY-MM-DD

## 变更类型
（基础配置 / 需求变更 / 技术架构 / 里程碑 / Bug修复 / 其他）

## 变更摘要
（一句话说明发生了什么）

## 变更详情
（具体改动了什么文件的什么内容，或完成了什么）

## 影响范围
（哪些模块/页面/接口/规则受到影响）

## 当前状态
（变更后的项目状态快照：当前阶段、主要待完成项、已知问题等）
```
